# Changelog

All notable changes to Pharos are documented here. This is the repository-level changelog;
per-package changelogs are generated by [Changesets](https://github.com/changesets/changesets)
under each package as they are versioned and published.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and the
project aims to follow [Semantic Versioning](https://semver.org/spec/v2.0.0.html). Pharos is
pre-1.0: minor versions may include breaking changes until 1.0.

## [0.1.0] — 2026-07-21

The initial tagged prototype. Ten build sprints (Bedrock → Signal) took Pharos from nothing
to a single, self-hostable platform where an agent action receives a policy verdict and
produces a signed, hash-chained, externally-verifiable evidence record in one transaction.
See [`docs/LIMITATIONS.md`](docs/LIMITATIONS.md) for the components still implemented as
honest stand-ins (linear judges, local KMS, simulated TSA, in-memory gateway held state) and
the roadmap tasks that replace them.

### Added

- **Bedrock** — the single-transaction invariant: a verdict and its sealed evidence record
  are two outputs of one write; both survive restart; the chain verifies genesis→head; the
  frozen `ActionRecord` schema v1 with a migration path.
- **Gatehouse** — enterprise SSO (OIDC, verified against Okta + Entra), scoped/rotatable API
  keys, deny-by-default RBAC, hard multi-tenant isolation (Postgres RLS under a
  `NOBYPASSRLS` app role + per-tenant signing keys), and a hash-chained access audit.
- **Lantern** — a tiered decision cascade (deterministic rules → statistical risk → served
  judge models, today linear bag-of-words classifiers) under a hard 800ms deadline with
  engineered fail-open/fail-closed semantics and a replay harness.
- **Causeway** — TypeScript + Python SDKs (deadline-aware, retries, local fail-mode),
  framework middlewares (LangChain/LangGraph, OpenAI Agents, Anthropic SDK, CrewAI, MS Agent
  Framework) under one conformance contract, a zero-code HTTP egress gateway, programmatic
  mandates, and exactly-once escalation resume.
- **Watchroom** — review operations: a routing queue engine, a deadline-aware SLA engine with
  exactly-once breach alerts, audited multi-channel notifications, reviewer analytics, and a
  disagreement→draft-rule feedback loop.
- **Seal** — legally-usable evidence: trusted-time anchoring (simulated TSA today), field-
  level redaction via selective disclosure, litigation hold, audience-scoped claims packs
  verified offline from the bundle alone, and regulatory exports (FINRA / EU AI Act Art. 12 /
  SR 11-7).
- **Codex** — citation-level FINRA and HIPAA regulation packs v2, a constrained-grammar
  policy compiler (v1), and a full policy lifecycle (compile → dry-run → shadow → active →
  sub-minute rollback).
- **Beam Count** — a continuous assurance engine reporting a measured Wilson-score verified-
  accuracy lower bound, a unified risk profile, a release-blocking readiness gate, and a
  consent-gated underwriter feed.
- **Granite** — observability (Prometheus/OTel-style tracing/alerting), resilience (multi-AZ,
  documented RPO/RTO, a zero-evidence-loss region-failover exercise), customer-hosted GA
  (hardened Compose + Helm, CPU-only judge), and metering/billing that reconciles to usage.
- **Signal** — the open PDP specification v1.0 with a conformance suite and an independent
  reference implementation, a public `POST /v1/pdp` endpoint returning a signed evidence
  binding, and identity-rail integration.

### Notes

- 157 TypeScript tests + 10 Python tests run against real Postgres / S3 WORM (MinIO) / Redis;
  CI fails the build if integration tests skip.
- Open-source foundation (this release line): Apache-2.0 license, security/contribution
  policy, a real ESLint gate, and Changesets-based versioning.
